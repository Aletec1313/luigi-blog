¿Qué es Exactamente el Reasoning Fine-Tuning?

Imagina que estás enseñando matemáticas a un niño. Podrías mostrarle solo las respuestas finales ("2+2=4"), pero eso no le enseña a pensar. Lo mejor es mostrarle el proceso: "tengo 2 manzanas, añado otras 2, ahora cuento 1-2-3-4, ¡son 4!".

El Reasoning Fine-Tuning hace exactamente eso con los LLMs:

  • Chain-of-Thought (CoT): El modelo aprende a generar "pensamientos intermedios" antes de dar la respuesta final
  • Self-consistency: Genera múltiples caminos de razonamiento y elige el más consistente
  • Step-by-step verification: Verifica cada paso antes de continuar

La magia está en que los modelos medianos (7B-13B) pueden superar a modelos mucho más grandes en tareas de razonamiento si se entrenan correctamente. Un Qwen2.5 de 7B con fine-tuning de razonamiento puede humillar a un Llama3 de 70B en problemas matemáticos complejos.

El Desafío del Hardware: Correr IA en tu Mac

Apple Silicon es mágico para inferencia local, pero el fine-tuning es otra historia. Necesitas:

  • Mac Mini M2 Pro/M4 o superior (mínimo 16 GB RAM unificada)
  • Preferiblemente 32 GB o más para modelos 13B con batch sizes decentes
  • Almacenamiento SSD rápido (los datasets de entrenamiento ocupan espacio)

¿Tienes solo 8 GB de RAM? No te preocupes, aún puedes hacer fine-tuning con:

  • Modelos más pequeños (3B-4B parámetros)
  • LoRA con rank ultra-bajo (r=8 o menos)
  • Cuantización extrema (Q4_K_M o incluso Q3)
  • Batch size de 1 (entrenamiento lento pero funcional)
Mi setup real: Mac Mini M2 Pro con 16 GB RAM entrena un modelo 7B con LoRA (r=32, alpha=64) en ~2-3 horas para datasets de 1,000 ejemplos. ¡Funciona!

Métodos de Fine-Tuning: LoRA, QLoRA y DoRA

No necesitas entrenar todos los parámetros del modelo (eso serían miles de millones). Usamos técnicas de Parameter-Efficient Fine-Tuning (PEFT):

🔧 LoRA (Low-Rank Adaptation)

En lugar de modificar los pesos originales, LoRA añade matrices pequeñas ("adapters") a ciertas capas. Resultado: solo entrenas el 1-2% de los parámetros totales.

# Configuración típica LoRA para razonamiento lora_config = { "r": 32, # Rank: cuanto mayor, más capacidad de adaptación "lora_alpha": 64, # Escalado: generalmente 2x el rank "target_modules": ["q_proj", "k_proj", "v_proj", "o_proj"], "lora_dropout": 0.05, # Regularización "bias": "none" } # Total parámetros entrenables: ~0.5% - 1%

⚡ QLoRA (Quantized LoRA)

El modelo base se carga en 4-bit (usando libraries como BitsAndBytes), pero LoRA opera en 16-bit o 32-bit. Ahorras ~70% de memoria sin perder casi calidad.

🎯 DoRA (Weight-Decomposed Low-Rank Adaptation)

La evolución de LoRA que descompone los pesos en magnitud y dirección. Mejor para tareas complejas de razonamiento donde dirección y magnitud importan por separado.

Para razonamiento: Recomiendo DoRA si tu hardware lo permite, o LoRA con r=32+ si necesitas ahorrar memoria.

Los Datos: Cocinando el Dataset Perfecto

El secreto del reasoning fine-tuning está en los datos. Necesitas ejemplos que muestren paso a paso cómo llegar a la respuesta.

📝 Formatos de Datos Efectivos

1. Chain-of-Thought Explícito:

{"instruction": "Resuelve: Si un tren viaja a 60 km/h y otro a 80 km/h en sentido opuesto desde 280 km de distancia, ¿cuándo se encuentran?", "input": "", "output": "Primero, calculo la velocidad relativa: 60 + 80 = 140 km/h (se acercan). Luego, tiempo = distancia / velocidad = 280 / 140 = 2 horas.\n\nLos trenes se encuentran después de 2 horas."}

2. Razonamiento Multi-Paso con Verificación:

{"instruction": "Encuentra el área de un círculo con radio 5 cm", "reasoning": "Paso 1: Identificar fórmula A = π × r². Paso 2: Sustituir r = 5. Paso 3: Calcular 5² = 25. Paso 4: Multiplicar π × 25 ≈ 78.54. Verificación: Unidades correctas (cm²), valor razonable.", "answer": "Aproximadamente 78.54 cm²"}

📊 Datasets Recomendados (o crea el tuyo)

  • OpenThoughts-114k: Razonamiento matemático y científico
  • Code-290k-DeepSeekR1: Razonamiento en programación
  • Bespoke-Stratos-17k: Razonamiento de sentido común
  • NuminaMath-CoT: Especializado en matemáticas competitivas

Para crear tu propio dataset:

  1. Generación asistida: Usa Claude o GPT-4 para generar pasos de razonamiento, luego revisa manualmente
  2. Anotación manual: Más lento pero con mayor calidad
  3. Síntesis de código: Extrae patrones de razonamiento de repositorios de algoritmos
💡 Pro tip: Prefiero calidad sobre cantidad. 500 ejemplos bien estructurados > 10,000 ejemplos genéricos. El razonamiento es sobre patrones de pensamiento, no sobre memorización.

Implementación Paso a Paso con MLX

MLX es el framework de Apple para machine learning en Apple Silicon. Es increíblemente eficiente porque aprovecha la memoria unificada (RAM compartida entre CPU y GPU). Aquí te dejo un script completo:

🛠️ Instalación y Setup

# Crear entorno conda (recomendado) conda create -n mlx-reasoning python=3.11 conda activate mlx-reasoning # Instalar dependencias pip install mlx-lm transformers datasets peft

📝 Script de Entrenamiento Completo

import mlx.optimizers as optim from mlx_lm import load, generate, train from mlx_lm.lora import LoRALinear from datasets import load_dataset import json # 1. Cargar modelo y tokenizer model, tokenizer = load("Qwen/Qwen2.5-7B-Instruct") # 2. Preparar dataset con reasoning def format_example(example): return { "text": f"<|im_start|>user\n{example['instruction']}<|im_end|>\n<|im_start|>assistant\n{example['reasoning']}\n\nPor tanto: {example['answer']}<|im_end|>" } dataset = load_dataset("mlabonne/Openthoughts_reasoning", split="train") dataset = dataset.map(format_example) dataset = dataset.train_test_split(test_size=0.1) # 3. Configurar LoRA lora_config = { "r": 32, "lora_alpha": 64, "target_modules": ["q_proj", "v_proj"], # Capas más importantes para reasoning "lora_dropout": 0.05, "bias": "none" } # 4. Configurar entrenamiento training_args = { "learning_rate": 2e-4, "num_epochs": 3, "batch_size": 4, "gradient_accumulation_steps": 4, "warmup_steps": 100, "weight_decay": 0.01, "logging_steps": 10, "save_steps": 100, "output_dir": "./reasoning-lora-7b" } # 5. Entrenar print("🚀 Iniciando fine-tuning de razonamiento...") train( model=model, tokenizer=tokenizer, dataset=dataset["train"], val_dataset=dataset["test"], args=training_args, lora_config=lora_config ) print("✅ Entrenamiento completado!")

⚙️ Parámetros Clave Explicados

  • learning_rate (2e-4): Para LoRA usamos tasas más altas que fine-tuning completo. El rango 1e-4 a 5e-4 funciona bien.
  • r=32: Rank suficiente para capturar patrones de razonamiento complejos. Puedes subir a 64 si tienes VRAM de sobra.
  • gradient_accumulation_steps=4: Simula batch_size=16 usando solo batch_size=4 en memoria. Esencial para modelos grandes.
  • warmup_steps=100: Aumenta gradualmente el learning rate al inicio para estabilidad.

import mlx.core as mx from mlx_lm import load, generate def evaluate_reasoning(model_path, test_prompts): model, tokenizer = load(model_path) results = [] for prompt in test_prompts: # Generar CON y SIN reasoning activado response_cot = generate( model, tokenizer, prompt=prompt, max_tokens=512, temperature=0.1, # Baja para tareas objetivas verbose=False ) # Extraer respuesta final y pasos de razonamiento results.append({ "prompt": prompt, "response": response_cot, "has_thinking_steps": "Paso" in response_cot or "Step" in response_cot }) return results # Prompts de prueba test_prompts = [ "Si tres obreros tardan 4 días en construir una pared, ¿cuánto tardarán 6 obreros?", "Encuentra el siguiente número en la secuencia: 2, 6, 12, 20, 30, ?", "Un tren de 150m viaja a 90 km/h. ¿Cuánto tarda en cruzar un puente de 350m?" ] results = evaluate_reasoning("./reasoning-lora-7b", test_prompts)

✅ Lista de Verificación de Calidad

  • ⬜ ¿Genera pasos intermedios explícitos?
  • ⬜ ¿Mantiene coherencia lógica en secuencias largas?
  • ⬜ ¿Las conclusiones siguen razonablemente de los pasos anteriores?
  • ⬜ ¿Evita responder instántaneamente a problemas complejos?
  • ⬜ ¿Señala incertidumbres o suposiciones?

Si tu modelo pasa al menos 4 de 5, ¡tienes un razonador sólido!

Aplicaciones Prácticas del Mundo Real

¿Por qué molestarse en entrenar tu propio modelo de razonamiento? Aquí van casos reales donde brilla:

🏢 1. Asistente de Análisis Financiero Personal

En lugar de responder "Invierte en este fondo", tu modelo razona:

"Paso 1: Analizando tu perfil de riesgo (edad 35, horizonte 20 años) → tolerancia media-alta. Paso 2: Considerando la cartera actual (demasiado concentrada en un sector). Paso 3: Evaluando correlaciones de mercado. Conclusión: Diversificar 30% en índices globales reduce riesgo sin sacrificar retorno esperado."

💻 2. Debugger Inteligente de Código

No solo dice "hay un error", sino que razona:

"Línea 42: Función recursiva sin caso base. Paso 1: Identificando la condición de salida que falta. Paso 2: Analizando el stack de llamadas potencial. Paso 3: Sugerir base case if n <= 0: return 0."

📋 3. Planificación de Proyectos Complejos

Análisis de dependencias, caminos críticos, y asignación óptima de recursos razonando paso a paso en lugar de "generar un plan". Ideal para freelancers gestionando múltiples clientes.

🎓 4. Tutor Educativo Personalizado

Explica conceptos adaptándose al nivel del estudiante, mostrando exactamente dónde se atascó en su razonamiento. Por ejemplo, para un estudiante que no entiende por qué 0! = 1:

"Paso 1: Definición de factorial: n! = n × (n-1)!. Paso 2: Para n=1: 1! = 1 × 0!. Paso 3: Pero 1! = 1, entonces 1 = 1 × 0!. Paso 4: Por tanto, 0! debe ser 1 para que la ecuación funcione."

🧪 5. Investigador Científico Automatizado

Analiza papers, genera hipótesis, y diseña experimentos paso a paso. Por ejemplo, al analizar un estudio sobre eficiencia de paneles solares:

"Paso 1: Identificar variables clave (eficiencia, temperatura, material). Paso 2: Analizar correlaciones en datos brutos. Paso 3: Detectar posible sesgo en condiciones de prueba (solo días soleados). Paso 4: Sugerir experimento complementario con condiciones variables."
